Reference-frame models Monty built

This is what recognition runs against: for each object, Monty stores a 3D graph of surface locations in an object-centered reference frame. Recognition is matching a moving sensor's observations into one of these frames. Notice the golf ball is a near-perfect sphere and the bowl a symmetric shell — the same models whose orientation is hardest to pin down.

Evidence accumulating in reference frames

One recognition episode, step by step. Each faint line is a candidate object’s evidence; the bold line is the true object (mug). Early on all ten are bunched and the correct one isn’t even leading — then, as the sensor moves and observations land in the object’s reference frame, competitors are eliminated and the target’s evidence climbs alone past threshold. This is the Thousand Brains algorithm converging.

Reproducibility

Variance across independent runs

Five independent seeds (0–4). By the benchmark’s own metric (correct incl. most-likely-hypothesis) every run scores 100% — an exact match to the documented 100.00%, independently corroborated by match-steps (30=30) and rotation error (12.85° vs ~13°). Strict convergence (excluding the MLH fallback) is 99.2% ± 1.17, with 0 misrecognitions in 500 episodes. (An earlier “95%” baseline here was a page-summary error; corrected against the raw table.)

  • This benchmark samples rotations randomly per run, so a single run vs the published 100% is sampling variance — hence the N=5 study above.

From reproduction to research

Growing the architecture — first results

The reproduction is the baseline. The research question is whether the cortex's architecture can be grown, not hand-designed. The first experiments — evolving the voting topology on the fixed 5-column model — give a pre-registered, inconclusive result: on the shipped tasks we can claim neither that voting topology matters nor that it doesn't (10-object is ceiling-saturated; 77-object is noise-limited even at 10 seeds). The one real, tool-grounded finding is zero-mean — a sparse tree topology genuinely helps on some object poses and hurts on others, cancelling in the mean.

The real win here is the method — a paired design whose adversarial review caught a verdict-flipping analysis bug before the data landed — plus a round-trip-verified generator that mints a valid N-column cortex for any N. Read the results & the new challenges →

Since then, three more results. Growing the column count found a grown three-column cortex accuracy-equivalent to the designed five-column one at ~⅓ the compute (the frontier →). Re-analysing that data explained why accuracy never rose with column count: the columns make their errors on the same objects (φ ≈ 0.75), so more of them buy redundancy, not capability (the correlation bottleneck →). That negative is a statement about a saturated task — so the programme moved to a world the agent changes, where an object-factored model plans on novel arrangements a monolithic learner can't match with 10× the data: count buys nothing, but structure pays (structure, not count →).
